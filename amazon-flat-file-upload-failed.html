Amazon Flat File 上传失败后的排查流程

批量上传失败后，不要立刻重传完整文件。先下载 Processing Report，按错误码、SKU、原始行号和字段名定位问题。

Updated: 2026-05-30 - Independent seller utility, not affiliated with Amazon.

不要立刻重传，先做这 5 步

  1. 下载 Processing Report。先确认失败记录、成功记录和错误总数，不要只看 Seller Central 页面上的红色状态。
  2. 按错误码分组。找出影响最多 SKU 的错误码，通常先修 8541、8560、99010、90244、8036。
  3. 按字段分组。如果多个 SKU 指向同一个字段，优先修模板列名、Valid Values 或条件必填组。
  4. 回原始行号修复。用 original-record-number 回到原始文件，不要在处理报告里直接改。
  5. 小批量复传。先用 1-5 个 SKU 验证修复方向，再批量上传整份文件。

上传失败排查树

现象优先判断下一步
所有 SKU 都失败模板站点、类目、文件格式或必填字段组可能错误检查模板来源、Data Definitions、Required 字段和保存格式。
某个父子体组失败parentage、parent_sku、relationship_type、variation_theme 不一致先看 父子体关系指南。
同一字段大量失败字段值不在允许列表，或列名/枚举值不匹配按字段分组，核对 Valid Values。
少数 SKU 失败商品编码、目录状态、ASIN 匹配或媒体 URL 个别问题按 SKU 分组，逐个核对报告 message。

示例报告行

original-record-number	sku	error-code	error-type	error-message	attribute-name
57	PARENT-MUG-BLK	99010	Error	A value is missing from one or more required columns from this group: [variation_theme = null].	variation_theme
64	MUG-BLK-12	90244	Error	We can't accept the Count you entered for Unit Count.	unit_count
77	MUG-BLK-IMG	20000	Error	We could not access the media at URL https://example.com/image.jpg.	main_image_url

重新上传前 checklist

  • 是否使用目标站点、目标类目的最新模板。
  • 是否保留原始列头，未删除 Data Definition 要求的必填列。
  • 父体和子体是否使用同一个 variation_theme。
  • 所有子体是否有唯一的商品编码、价格、库存和变体属性组合。
  • 枚举字段是否来自 Valid Values，而不是手动翻译或复制旧值。
  • 图片 URL 是否可公网访问，且不是登录后才可访问。

边界说明

本页用于上传后排错，不保证 Seller Central 一定接受修改后的文件。最终通过状态取决于站点、类目、品牌权限、账号权限、模板版本和 Amazon 当前目录规则。
