Amazon Flat File 上传失败后的排查流程

批量上传失败后，不要立刻重传完整文件。先看 Processing Report，找到错误码、原始行号和字段名。

三步定位

  1. 下载 Processing Report，确认是否有 error-code 和 error-message。
  2. 按错误码分组，先处理影响 SKU 最多的问题。
  3. 回到原始模板按 original-record-number 定位行号。

先修哪些错误

优先处理 8541、8560、90244、99010，因为它们通常会阻断 SKU 创建或变体建立。图片、长度、格式类问题可以第二批处理。
