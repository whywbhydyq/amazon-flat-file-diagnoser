什么时候用这个分诊流程？

Size / color / style 变体无法归组适合服饰、鞋包、珠宝、饰品和 POD 商品，尤其是 child ASIN 被拆成独立 listing 或 dropdown 不显示。 Processing Report 已有 8008 / 8036 / 8058 / 90244 / 99010这些错误常和 parentage、parent_sku、relationship_type、variation_theme 或变体属性有关。 VA / Agency 需要先给客户出诊断先生成 SKU-level 问题表，再决定是否生成修正版 flat file 或转人工修复。 试过 Seller Central 仍不知道错在哪分清是 flat file 结构错误、枚举值错误、catalog conflict，还是 Seller Support 才能处理的问题。

工具会检查哪些字段？

字段组检查内容为什么重要
sku / parent_skuchild 是否指向真实存在的 parent SKU，是否有 orphan child。父子体无法关联时，常见原因是 SKU 引用错误或父体行缺失。
parentageparent 行和 child 行是否区分正确。父体通常是分组行，子体才是真实可售 SKU。
relationship_typechild 行是否填写 variation 或模板要求的关系值。关系字段缺失会导致变体关系 feed 失败。
variation_theme同一 family 内是否一致，是否缺失。theme mismatch 会让 size/color/style 关系无法成立。
size_name / color_name / style_namechild 是否有对应变体属性，组合是否唯一。dropdown 不显示或属性值被拒，通常要从这里查。

分诊结果如何解释？

结果含义下一步
File-fixable低风险空字段或可从同一文件确认的结构问题，例如 relationship_type 缺失、可推断的 variation_theme 缺失。先预览 safe draft diff，再下载修复草稿；不自动补无法确认的 parent_sku。
Needs variation triageProcessing Report 只能说明报错，必须结合 flat file 才能判断结构原因。补充 flat file，生成 family map。
Needs manual catalog review可能涉及 ASIN contribution、品牌、类目限制、现有 catalog 锁定。人工判断，不承诺单靠文件修复。
Seller Support likely文件结构看起来正常，但 catalog 状态或后台限制导致无法更新。整理 SKU、错误码、字段和截图，准备 case。

Safe repaired-file draft 是什么？

当 flat file 中存在低风险、可从同一文件推断的空字段时，工具会生成一个本地修复草稿。例如 child 行缺少 relationship_type，或同一 family 已经有唯一的 variation_theme 但部分行为空。

草稿不会自动推断缺失的 parent_sku，不会清空 parent 行的 price / quantity，不会创建新 SKU，也不会处理 8541、8036、Brand Registry、catalog contribution 或 Seller Support 才能解决的问题。

第一版不会做什么？

  • 不要求 Seller Central 登录，不接账号权限。
  • 不自动上传到 Amazon，不绕过平台流程。
  • 不保证恢复所有 parent-child 关系，因为 catalog-level 问题不一定能靠 flat file 修复。
  • 不处理 Brand Approval、Hazmat / SDS、Listing SEO 文案优化。
