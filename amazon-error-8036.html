Error 8036

Amazon 8036 错误：变体关系创建受限

8036 不应再按“商品编码无效”处理。Amazon 官方说明它通常与创建 ASIN 变体关系的能力被移除有关，常见提示会提到 policy violations、variation relationships 或 30 days。

Updated: 2026-05-30 - Independent seller utility, not affiliated with Amazon.

优先检查提示类型

先看 Processing Report 的 error-message 是否明确提到变体关系、政策限制或 30 天限制；如果是账号/政策限制，单纯改字段通常无法立即解决。

修复步骤

  1. 确认 Processing Report 原文是否提到 variation relationships、policy violations 或 30 days。
  2. 检查 parent_sku、parentage、relationship_type、variation_theme 是否一致，但不要把账号限制误当成字段拼写错误。
  3. 准备 batch id、case id、父/子 SKU、ASIN 和完整 Processing Report，必要时联系 Seller Support 或 Account Health。
  4. 如果仍需修字段，先用一个父体和 1-2 个子体做最小复现文件。

字段定位表

先用 Processing Report 的 error-message、attribute-name 和 original-record-number 回到原始 Flat File 行，再按下表逐列检查。

字段检查重点修复动作
parent_sku确认子体引用的父 SKU 是否存在拼写一致，不要引用不存在或未成功创建的父体
parentage父体为 parent，子体为 child不要把父体/子体行混填
relationship_type确认当前模板允许的关系类型使用模板中的关系值，不自行翻译
variation_theme父体和所有子体必须一致颜色/尺寸等变体属性要和 theme 对齐
error-message确认是否为政策限制如果是限制类提示，优先走官方支持流程

重新上传前 checklist

  • 只保留 1-3 个受影响 SKU 做最小复现文件，避免整表反复失败。
  • 确认模板来自当前 marketplace、当前商品分类和最新版 Seller Central。
  • 不要只根据错误码猜原因；优先读取 error-message 和 attribute-name。
  • 重新上传后下载新的 Processing Report，确认同一字段不再报错。
